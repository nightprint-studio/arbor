Arbor Showcase

A comprehensive reference plugin that exercises every Arbor plugin API surface in one place. Use it to verify that the plugin system is working correctly, or as a living example when building your own plugins.

What it does

  • Tracks a persistent load counter in global settings — incremented every time Arbor starts
  • Adds a sidebar panel toggle ("Showcase") in the Activity Bar; opening the panel shows a preferences form
  • Adds a quick action button (⚡) in the Activity Bar bottom section
  • Adds context-menu items on commits and branches
  • Adds an "Open Showcase Panel" entry to the hamburger menu
  • Reacts to on_commit, on_push, on_checkout, on_fetch, and on_repo_open with toasts and log entries
  • Runs a background scheduler every 60 seconds that logs a tick and optionally sends a persistent notification

on_checkoutShows a "Branch checkout detected!" toast and logs the new branch

Settings

Open Plugin Manager → gear icon or the sidebar Showcase panel to adjust:

SettingDefaultDescription
Your name(empty)Stored in global settings for demonstration of text persistence
Enable scheduled notifications✓ onWhen on, each 60-second scheduler tick sends a persistent notification to the bell overlay
Custom Labels(empty)Key-value pair list — demonstrates the kv_list form field type

Sidebar panel

Click the ⚡ Showcase icon in the Activity Bar to open the preferences form directly in a modal. The form demonstrates three layout sections and multiple field types including text, select, and checkbox.

Using this as a template

Every Arbor plugin API feature used in main.lua is annotated with a comment. Search for the API call you want to use and copy the surrounding pattern:

arbor.ui.add_sidebar()        ← activity bar sidebar toggle (left or right rail)
arbor.ui.add_context_menu_item()← commit / branch context menu
arbor.ui.add_menu_item()       ← hamburger menu entry
arbor.ui.form()              ← modal input form
arbor.notify{ ... }          ← persistent notification
arbor.settings.global.*      ← cross-session key-value storage
arbor.job.spawn()            ← background process with streaming output
arbor.scheduler.register()   ← Spring-style background scheduler (fixed_rate / fixed_delay / cron)
