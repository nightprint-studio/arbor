Panel Demo

Minimal proof-of-concept for the right-side ActivityBar and the arbor.ui.add_sidebar + arbor.ui.set_panel_content API. When the plugin is enabled, Arbor reveals the right-side icon rail with two entries:

  • 🧩 Panel Demo — right sidebar (top area). Opens a panel with a heading, a counter that proves the panel:open:<id> hook re-fires on every reopen, a list of clickable actions, and a footer button.
  • 📋 Panel Demo — bottom — right bottom area. Opens the unique bottom panel; clicking it while Stage / Terminal / Pipelines is open swaps the content (same slot).

What this plugin demonstrates

  • arbor.ui.add_sidebar({id, side, position, ...}) — declare a right-side entry. side="right" + position="top" puts the icon in the top group; position="bottom" routes to the shared bottom panel slot.
  • panel:open:<id> — lifecycle hook fired every time the user opens the panel. The plugin is expected to respond with arbor.ui.set_panel_content.
  • arbor.ui.set_panel_content(id, {title, nodes, actions?}) — pushes form-DSL content into the panel. Supported node types in the lightweight sidebar renderer: heading, label, paragraph, divider, button, list, section.
  • Permission surface — zero. The plugin declares git_read = false, git_write = false, filesystem = "none".

Uninstall

Disable or delete the plugin folder to remove both icons. The right ActivityBar auto-hides when no plugin is contributing — the layout falls back to the classic IntelliJ-left-only style.
