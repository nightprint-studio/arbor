Branch Stats

A minimal, fully-commented reference plugin that demonstrates the Arbor plugin API. It logs branch lifecycle events to the Arbor console, registers UI elements across multiple surfaces, and shows a commit annotation form — all in a single easy-to-read file.

What it does

  • Logs every commit, checkout, and fetch event to the Arbor plugin log
  • Shows a toast on commit and fetch events as a simple confirmation
  • Adds a sidebar panel toggle ("Branch Stats") in the Activity Bar top section
  • Adds "Inspect Commit" to the commit right-click context menu — opens a form where you can add a personal note, a tag, and a bookmark flag to any commit
  • Adds "Branch Stats: Run Report" to the hamburger menu

Requirements

PermissionWhy
git_read = trueReads branch and commit information from hook context

No network or filesystem access is needed.

Hooks

HookAction
on_plugin_loadLogs a "ready" message with the API version
on_commitLogs commit OID + branch; shows a success toast
on_checkoutLogs the new and previous branch name
on_fetchShows an info toast confirming the fetch

Inspect Commit form

Right-click a commit → Inspect Commit to open a form with three fields:

  • Note — free-text note about the commit
  • Tag — a short label (e.g. "performance", "fix", "refactor")
  • Bookmark — checkbox to bookmark the commit

Submitting saves the note to the plugin log. This is a demonstration of the form API and state echo — in a production plugin the data would be persisted with arbor.settings.project.set().

Use as a starting point

Branch Stats is intentionally simple. Clone it as a starting point for your own plugin:

plugins/
  my-plugin/
    plugin.toml   ← copy from branch-stats/plugin.toml, change name
    main.lua      ← copy from branch-stats/main.lua, adapt hooks
