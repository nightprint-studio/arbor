vertex-branding

Sample enterprise-branding plugin. Demonstrates every runtime branding surface Arbor exposes by re-skinning the app with the (fictional) Vertex Systems identity. Disabling the plugin reverts every change — nothing is persisted to disk.

What gets re-skinned

  • App mark (in-app SVG): the rounded V logo paints in the title-bar slot left of the hamburger, in the welcome screen hero, in the About modal header — and is embedded by the HTML stats exporter so any report you generate stays co-branded.
  • OS window icon (raster): the same mark, exported as assets/vertex.png, is handed to the OS via Tauri's WebviewWindow::set_icon. Updates the taskbar entry and the Alt-Tab thumbnail on Windows / Linux. macOS dock icons come from Info.plist and require a build-time swap, so this field is a no-op on macOS — the in-app mark still applies.
  • Accent palette: a CSS-variable overlay is layered on top of the active theme via arbor.ui.set_theme_tokens. Switching themes (dark ⇆ light) keeps the orange accent — the overlay is reapplied on every theme change.

Setup

  1. Generate assets/vertex.png from assets/vertex.svg. Any vector tool works — Inkscape, Illustrator, online converters, or one-liner CLI tools (e.g. magick vertex.svg -resize 256x256 vertex.png). 256×256 is a sensible target for taskbar icons; on Windows you can also ship a multi-resolution .ico and point window_icon_path there instead.
  2. Enable the plugin in the Plugin Manager.
  3. A toast confirms the switch. Pop the title-bar palette and the welcome screen open — both should be Vertex orange.

If the PNG is missing when the plugin loads, the SVG mark is still applied and a warning is recorded in the Plugin Logs panel. Drop the PNG in place and reload the plugin to pick up the window-icon swap.

Files

FilePurpose
plugin.tomlManifest. Declares the two lifecycle hooks and zero permissions — the branding APIs are unrestricted.
main.luaEmbeds the SVG inline, builds the absolute icon path from arbor.meta.plugin_dir(), and applies / clears the overrides on the lifecycle hooks.
assets/vertex.svgSource artwork. Single source of truth — the Lua file's inline copy is regenerated from this. Used to produce the PNG.
assets/vertex.pngRaster window icon (you generate this from the SVG).

Permissions

None. set_branding / clear_branding / set_theme_tokens / clear_theme_tokens are unrestricted UI APIs (same trust model as arbor.notify), the SVG is inlined in main.lua so no arbor.fs.read is needed, and the PNG path is loaded server-side by the backend through Tauri's image API — outside the plugin's filesystem sandbox.

Adapting it for your own brand

  1. Replace the VERTEX_SVG long-string in main.lua with your own inline SVG. Keep viewBox="0 0 512 512" for a square mark; non-square marks render fine but center themselves in the in-app slots.
  2. Drop your raster icon at assets/<your-name>.png (or .ico) and update ICON_PATH accordingly.
  3. Tune the THEME_OVERLAY table. Any CSS variable defined by Arbor's design system can be overridden — see the Themes & Presets docs page for the full list.
