GitHub Actions

Overlays GitHub Actions CI status on commits in the graph. After every fetch, the plugin queries the GitHub Checks API and caches the run result per commit SHA — so you can see at a glance whether a build passed or failed without leaving Arbor.

What it does

  • Parses the active repository's remote URL to extract the owner/repo slug (supports HTTPS and SSH remotes)
  • Maintains an in-memory cache of oid → { status, updated_at }
  • Invalidates the cache entry for the latest commit on every on_commit event, so the new run is picked up on the next fetch
  • Refreshes CI statuses from the GitHub Checks API on on_fetch

Requirements

PermissionWhy
network = ["api.github.com"]Queries the GitHub Checks / Commit Statuses API to fetch CI run results
git_read = trueReads the active repository path and remote URL

Hooks

HookAction
on_commitClears the cache for the new commit OID so the next fetch picks it up
on_checkoutLogs the branch switch for debugging
on_fetchBatch-requests the latest CI statuses from the GitHub API and updates the cache

Supported remote formats

# HTTPS
https://github.com/owner/repo.git

# SSH
git@github.com:owner/repo.git

Non-GitHub remotes are silently ignored — the plugin does nothing for GitLab, Bitbucket, or self-hosted repositories.

Notes

  • No API token is configured — requests are unauthenticated and subject to GitHub's rate limits (60 req/hour per IP). For private repositories or high-volume use, a future settings form will let you provide a Personal Access Token.
  • The cache is in-memory only and is cleared when Arbor is restarted.
