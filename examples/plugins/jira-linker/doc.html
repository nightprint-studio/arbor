JIRA Linker

Automatically detects JIRA issue keys (e.g. PROJ-1234, AUTH-42) in commit messages and builds an in-memory index so you can query which issues a commit is related to — without leaving Arbor.

What it does

  • On every commit, scans the commit message for patterns matching [A-Z][A-Z0-9]+-[0-9]+ (standard JIRA key format)
  • Maintains an in-memory index of commit OID → list of issue keys
  • Adds "Show JIRA Issues" to the commit right-click context menu — shows all detected keys for that commit in a toast
  • Fully offline — no network requests are made; all matching happens locally in Lua

Requirements

PermissionWhy
git_read = trueReads commit messages and OIDs from hook context

No network or filesystem permissions are needed — everything runs locally.

Detected key format

JIRA keys follow the pattern: one or more uppercase letters + dash + one or more digits.

feat(AUTH-12): add OAuth login     → detects AUTH-12
fix PROJ-99 and CORE-7             → detects PROJ-99, CORE-7
Merge pull request for BACKEND-301 → detects BACKEND-301

Context menu

Right-click any commit → Show JIRA Issues to see the detected issue keys for that commit. If no keys were found the plugin shows an "No JIRA keys found" info toast.

Settings

Open Plugin Manager → gear icon next to JIRA Linker to configure:

SettingDescription
JIRA Base URLYour JIRA instance root, e.g. https://yourcompany.atlassian.net. Used to construct issue deep-links in future versions of the plugin.

Notes

  • The index is in-memory only and is rebuilt as you make commits. Historical commits are not scanned automatically — open each one and run "Show JIRA Issues" from the context menu.
  • Duplicate keys in the same commit message are deduplicated automatically.
